## Architecture Overview

### Python Training Pipeline
```
arena.py          ← Main adversarial training loop (run this first)
export_onnx.py    ← Converts trained .pt weights → .onnx for browser
models.py         ← Encoder, Decoder, Warden CNN architectures
dataset.py        ← Image loader (real images or synthetic fallback)
metrics.py        ← SSIM, PSNR, BER, detection accuracy
visualize.py      ← Arms race plot (3-panel matplotlib chart)
inference.py      ← CLI: embed / detect / extract using trained weights
```

### Browser Demo
```
demo/index.html   ← Full client-side demo, ONNX Runtime Web, no backend
demo/encoder.onnx ← Generated by export_onnx.py (add after training)
demo/decoder.onnx
demo/warden.onnx
```

### Key parameters (must stay in sync across all files)
- `IMAGE_SIZE  = 64`   — training image resolution (arena.py, export_onnx.py, demo/index.html)
- `PAYLOAD_LEN = 32`   — bits per secret payload (arena.py, export_onnx.py, demo/index.html)
- `LAMBDA_IMAGE = 1.0`, `LAMBDA_DECODE = 1.0`, `LAMBDA_ADVERSARIAL = 0.5` — loss weights in arena.py

If you change IMAGE_SIZE or PAYLOAD_LEN in arena.py, you MUST update the same values in:
  - export_onnx.py (--image_size, --payload_len defaults)
  - demo/index.html (const IMAGE_SIZE, const PAYLOAD_LEN at top of <script>)

## Stack & Constraints

### Python side
- PyTorch (CPU-only) — no CUDA assumptions, ever
- No external training services, no cloud APIs
- Batch size default 4 — keep 4–8 for CPU compatibility
- All model weights saved as .pt files in ./output/

### Browser side (demo/index.html)
- Single self-contained HTML file — no separate CSS or JS files
- ONNX Runtime Web loaded from CDN: cdn.jsdelivr.net/npm/onnxruntime-web@1.19.0
- No React, no bundler, no build step — pure HTML/JS/CSS
- Color palette uses CSS variables matching crypto-lab dark theme:
  --bg:#0d1117  --surface:#161b22  --accent:#58a6ff  --green:#3fb950  --red:#f85149
- index.html must be served over HTTP (not file://) for WASM workers to function
- Never use localStorage or sessionStorage

### Git discipline
- Commit after each meaningful change with a clear message
- Never commit output/*.pt files (large binary weights) — add to .gitignore
- DO commit demo/*.onnx files — these are browser assets users need
- Keep __pycache__/ in .gitignore

## Workflow Phases

### Phase 1 — Training
```bash
python arena.py --image_dir ./images --rounds 30 --epochs_per_round 5
```
- Produces: output/encoder_final.pt, decoder_final.pt, warden_final.pt
- Produces: output/arms_race_plot.png, arms_race_history.json
- Commit message: "train: round {N} complete — warden_acc={X} BER={Y} SSIM={Z}"

### Phase 2 — Export
```bash
python export_onnx.py --weights_dir ./output --out_dir ./demo
```
- Produces: demo/encoder.onnx, decoder.onnx, warden.onnx
- Verify file sizes are reasonable (encoder ~700KB, decoder ~500KB, warden ~150KB)
- Commit message: "export: onnx models from round {N} training"

### Phase 3 — Demo verification
```bash
cd demo && python -m http.server 8080
# Open http://localhost:8080 and verify:
# - All three model status dots turn green
# - Embed tab produces a downloadable stego image
# - Detect tab outputs a probability bar
# - Extract tab recovers text from a stego image
```
- Commit message: "demo: verified embed/detect/extract with round {N} models"

### Phase 4 — GitHub Pages
- Source: main branch, /demo folder
- URL: systemslibrarian.github.io/steg-arena/demo/
- Commit message: "deploy: steg-arena demo live"

## Common Tasks

### Add a new hyperparameter experiment
1. Duplicate arena.py as arena_experiment_name.py
2. Change only the LAMBDA_* values at the top
3. Run with --output_dir ./output_experiment_name
4. Compare arms_race_history.json files to baseline
5. Commit with message: "experiment: {description of lambda change}"

### Improve model architecture
1. Edit models.py ONLY — never change training logic in arena.py for architecture experiments
2. Keep ConvBnRelu as the building block unless explicitly instructed otherwise
3. After any architecture change, delete output/ and retrain from scratch
4. Document the change in a comment above the class definition

### Update the browser demo UI
1. Edit demo/index.html ONLY
2. Never change IMAGE_SIZE or PAYLOAD_LEN in the JS without matching Python changes
3. Preserve the CSS variable palette — do not introduce hardcoded colors
4. Keep the scripture footer: "Whatever you do, do it all for the glory of God." — 1 Corinthians 10:31
5. Test by serving over HTTP — not by opening as a file

### Re-export after retraining
```bash
python export_onnx.py
git add demo/encoder.onnx demo/decoder.onnx demo/warden.onnx
git commit -m "export: updated onnx from new training run"
git push
```

## Metrics Interpretation Quick Reference

| Metric | Value | Meaning |
|--------|-------|---------|
| warden_acc | 0.50 | Encoder has completely fooled the Warden |
| warden_acc | 0.80+ | Warden is winning — stego is detectable |
| encoder_ssim | 0.95+ | Payload is imperceptible to human eyes |
| encoder_ssim | < 0.85 | Distortion is becoming visible |
| encoder_ber | < 0.10 | Payload recovering reliably |
| encoder_ber | 0.50 | Payload is lost — encoder not converging |
