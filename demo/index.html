How This Works

This demo runs three neural networks trained through an adversarial arms race — similar to a GAN — entirely in your browser. No data is ever sent to a server.


Encoder (Stegger)
Learns to hide a binary payload in a cover image as an imperceptible pixel residual. Its loss function has three components: image imperceptibility (MSE/SSIM), payload recoverability (BER), and adversarial evasion (fool the Warden). Nobody programmed how it hides data — it discovers this through 20+ rounds of training.


Warden (Detector)
A binary CNN classifier trained to distinguish clean images from stego images. It learns to detect whatever statistical fingerprint the Encoder leaves — fingerprints that may not match any named technique in the academic literature.


Decoder (Extractor)
Recovers the hidden payload from a stego image. Bit Error Rate (BER) measures how accurately it recovers each bit. BER < 0.1 means the payload survived embedding reliably.


The Arms Race
Training alternates: the Warden trains to detect the Encoder's output, then the Encoder trains to fool the updated Warden. Over 20+ rounds, both networks evolve. The Encoder may discover steganographic strategies that have no human-designed analogue.


Limitation
The model weights act as a shared key — Alice and Bob must have the same trained encoder/decoder pair. For real-world use, the payload should be AES-encrypted before embedding. Steganography hides it; cryptography protects it.

Architecture Reference

Based on HiDDeN (Zhu et al., ECCV 2018) — the foundational neural steganography architecture.

Trained locally with PyTorch (CPU). Exported to ONNX and served via ONNX Runtime Web — no backend, no server, no data leaves your browser.
